Help

With an increase of complex scientific software, researchers need to spend a large portion of their time searching for avaliable software documentation and source code, and reading it to understand the software used and the proper way to cite it. SoMEF is designed to help highlight the most significant aspects of the scientific documentation by processing README files in GitHub repositories. It automatically extracts the contents of the README file into 4 categories: description, installation, invocation, or a citation of a software component.

Frequently Asked Questions

What is the classifer threshold?

The classifer threshold is a threshold that we use in our classifers to determine whether a sentence should be classified as a description, installation, invocation or citation.

Is having a higher or lower classifer threshold better?

This is ultimately dependent on how strict you want the classifer to choose whether a sentence falls into a certain category. For example, a high classification threshold may cause certain sentences that are description sentences not be classified due to the small margin of error given by the threshold. On the other hand, if you were to give it a small classification threshold, it may cause sentences that are not description sentences to be classifed as such.

Am I required to give a classifer threshold?

If you do not provide a specific threshold, we will automatically use a threshold of 80%.
